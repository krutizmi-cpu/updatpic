# UpdatPic — правила проекта для людей и агентов

Прочитай этот файл в начале любой сессии по репозиторию `updatpic`.

## 1. Что это

`UpdatPic` — отдельный продукт для:

- поиска фото товара по артикулу, сайту поставщика и прямым ссылкам;
- скачивания и локального хранения изображений;
- повторного использования уже найденных фото;
- подготовки архивов под требования конкретных клиентов.

Проект не связан с `nsi` и не должен развиваться внутри репозитория НСИ.

## 2. Границы проекта

1. Репозиторий отвечает только за фото-контент и клиентские выгрузки.
2. Не переносить сюда бизнес-логику НСИ, PIM, маркетплейсных карточек целиком или складские процессы, если это не связано напрямую с фотографиями.
3. Источник поиска по умолчанию:
   - прямые ссылки менеджера;
   - сайт поставщика;
   - интернет-поиск как fallback.
4. Если фото нельзя надёжно получить автоматически, оставлять понятный статус и давать возможность догрузки вручную.

## 3. Канон данных

Основная сущность товара:

- `article` — внутренний артикул;
- `supplier_article` — артикул поставщика;
- `name` — название товара;
- `quantity` — количество/остаток, если есть;
- `supplier_site` — домен или сайт поставщика;
- `product_url` — прямая ссылка на карточку;
- `image_urls_raw` — прямые ссылки на картинки, возможен разделитель `;`.

Основная сущность изображения:

- исходный URL;
- локальный путь;
- источник поиска;
- порядок фото;
- размер файла;
- ширина и высота.

## 4. Клиентские правила

Правила именования и упаковки лежат в `data/client_profiles.json`.

На старте поддерживаются:

- `sportmaster`
- `detmir`

Новые клиенты добавляются конфигом в JSON, без переписывания базовой логики выгрузки.

## 6. Бизнес-правила, которые пока считаются обязательными

- Если менеджер дал прямые URL на фото, они должны обрабатываться в первую очередь.
- Если у поставщика есть ссылка на товар, её нужно пытаться разобрать раньше общего интернет-поиска.
- Уже найденные фото должны сохраняться локально и быть доступны для повторной клиентской выгрузки без нового парсинга.
- Клиентская выгрузка обязана переименовывать файлы по шаблону клиента.
- Контентные требования маркетплейсов считаются подсказкой и фильтром проверки, но не юридической гарантией прохождения модерации.

## 7. История проекта

### 2026-04-27

- Создан отдельный репозиторий `updatpic`.
- Определён первый сценарий: менеджер загружает товары, сервис ищет фото, затем формируется архив под клиента.
- Выбран стек `Streamlit + SQLite + requests + Pillow + pandas`.
- Добавлены стартовые профили клиентов для Спортмастера и Детского мира.
- Реализован базовый каркас UI, локальная база и первый pipeline скачивания изображений.

## 8. Definition of done для ближайшего этапа

- Можно загрузить таблицу товаров и получить сохранённые фото хотя бы для части позиций.
- Можно загрузить таблицу клиентских кодов и скачать ZIP-архив.
- Профили клиентов меняются через JSON.
- Источники и ограничения проекта описаны в `README.md` и этом файле.
