# UpdatPic

`UpdatPic` — сервис для поиска, скачивания, хранения и клиентской выгрузки фотографий товара. На входе обычно есть артикул, название, сайт поставщика, а иногда и готовые ссылки на фото.

## Что умеет

- Принимать Excel/CSV с товарами.
- Искать фото по прямым URL, странице поставщика, внутреннему поиску по сайту поставщика и интернет-поиску.
- Сохранять найденные изображения локально и связывать их с артикулом.
- Отдавать Excel-шаблоны для менеджера, Спортмастера и Детского Мира.
- Давать в интерфейсе AI-настройки для OpenAI-compatible провайдера, например NVIDIA NIM.
- Работать в разделе `Клиенты` по двум сценариям в одном файле:
  - `каталог`: фото уже спарсены и лежат в локальной базе;
  - `ссылки`: готовые URL уже есть, парсинг не нужен.
- Для `каталога` собирать ZIP с переименованными файлами.
- Для `ссылок` собирать Excel по формату площадки.
- Поддерживать Yandex OAuth 2.0 с подписанной cookie-сессией после настройки переменных окружения.

## Поток работы

1. Менеджер загружает файл товаров в раздел `Каталог фото`.
2. Сервис ищет и сохраняет фото по каждому товару.
3. В разделе `Клиенты` загружается единый клиентский файл.
4. Для каждой строки сервис определяет режим:
   - если есть только `article + код клиента`, берутся локальные фото;
   - если есть `код клиента + ссылки`, строится Excel со ссылками;
   - если в одном файле есть оба типа строк, сервис отдаёт оба результата.

## Поддерживаемые колонки менеджера

Обязательные:

- `артикул`
- `название`

Опциональные:

- `шт`
- `сайт`
- `артикул поставщика`
- `ссылка на товар`
- `картинки`
- `комментарий`

Для поля `картинки` поддерживаются разделители `;`, `,`, пробел и перенос строки.

## Клиенты

Профили лежат в `data/client_profiles.json`.

### Спортмастер

- Локальная выгрузка: ZIP с именами `КодЦветомодели_1`, `КодЦветомодели_2` и так далее.
- Форматы: `jpg`, `jpeg`, `png`
- Размер файла: до `50 МБ`
- Готовые ссылки: только прямые открытые URL на `jpg` / `jpeg` / `png`
- Разделитель ссылок: `;` без пробелов

Источник, проверен 28 апреля 2026 года:

- https://seller-help.sportmaster.ru/pages/viewpage.action?pageId=48365664

### Детский Мир

- Локальная выгрузка: ZIP с именами `Штрихкод_01`, `Штрихкод_02` и так далее.
- Форматы: `jpg`, `jpeg`, `png`, `webp`
- Длинная сторона: `1000-8000 px`
- Размер файла: до `1 МБ`
- Готовые ссылки: прямые URL на изображение или публичные ссылки Яндекс Диска
- Лимит: максимум `30` фото на товар

Источник, проверен 28 апреля 2026 года:

- https://help.detmir.market/edit-media

## Excel-шаблоны

После `node scripts/build_templates.mjs` появятся:

- `templates/manager_import_template.xlsx`
- `templates/sportmaster_upload_template.xlsx`
- `templates/detmir_upload_template.xlsx`

Что внутри:

- `manager_import_template.xlsx`:
  лист `Менеджер` для каталога и лист `Как использовать`.
- `sportmaster_upload_template.xlsx`:
  один шаблон под оба сценария с колонками `article`, `Код цветомодели*`, `Источник фото`, `Ссылки на фото`.
- `detmir_upload_template.xlsx`:
  один шаблон под оба сценария с колонками `article`, `Штрихкод товара*`, `Источник фото`, `Ссылки на изображения товара`.

Реальные примеры площадок учтены так:

- у шаблонов клиента есть строка-памятка над заголовками;
- сервис умеет сам находить строку заголовков, даже если первая строка — это инструкция площадки;
- один и тот же клиентский файл можно смешивать: часть строк из каталога, часть по готовым ссылкам.

## Yandex OAuth

Для включения входа через Яндекс задайте:

```bash
UPDATPIC_ENABLE_YANDEX_AUTH=1
UPDATPIC_REQUIRE_LOGIN=1
YANDEX_CLIENT_ID=...
YANDEX_CLIENT_SECRET=...
YANDEX_REDIRECT_URI=https://<ваш-домен-streamlit>
UPDATPIC_AUTH_SECRET=<случайный-длинный-секрет>
```

В текущей реализации UpdatPic:

- инициирует OAuth 2.0 `code` flow Яндекса;
- обменивает `code` на токен на серверной стороне;
- получает профиль пользователя через API Yandex ID;
- сохраняет собственную подписанную cookie-сессию до 30 дней, чтобы вход не слетал при обычном обновлении страницы.

Документы, использованные для настройки:

- Streamlit `st.login()` работает только с OIDC-провайдерами и хранит cookie в браузере: https://docs.streamlit.io/1.43.0/develop/api-reference/user/st.login
- Yandex ID описывает веб-вход через OAuth 2.0 `code` flow и `redirect_uri`: https://yandex.ru/dev/id/doc/en/codes/code-url
- Информация о пользователе запрашивается через `GET https://login.yandex.ru/info` с `Authorization: OAuth <token>`: https://yandex.ru/dev/id/doc/en/user-information

## AI-провайдер

Во вкладке `AI` можно:

- задать `base_url`
- ввести и сохранить `API key`
- переключить `model`
- проверить связку `ключ + модель` и увидеть явный статус, если ключ не работает

По умолчанию выставлен NVIDIA NIM с:

- `base_url = https://integrate.api.nvidia.com/v1`
- `model = deepseek-ai/deepseek-v3.2`

Опорные ссылки:

- NVIDIA DeepSeek V3.2 deploy page: https://build.nvidia.com/deepseek-ai/deepseek-v3_2/deploy
- NVIDIA OpenAI-compatible chat completions endpoint: https://docs.api.nvidia.com/nim/reference/deepseek-ai-deepseek-v3_2-infer

## Ограничения текущей версии

- Это рабочий каркас, а не финальный production-парсер.
- Интернет-поиск всё ещё завязан на HTML-выдачу DuckDuckGo и разбор страниц.
- Автоматической модерации контента фото под правила площадок пока нет.
- Нет фоновой очереди задач.
- Yandex OAuth требует ручной настройки переменных окружения и Redirect URI.
