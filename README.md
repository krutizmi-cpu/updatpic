# UpdatPic

`UpdatPic` — отдельный сервис для поиска, скачивания, хранения и клиентской выгрузки фотографий товара. Проект рассчитан на менеджерский поток, где на входе есть артикул, количество, название, сайт поставщика и иногда прямые ссылки на изображения через `;`.

## Что уже умеет стартовая версия

- Принимать Excel/CSV с товарами.
- Отдавать готовые Excel-шаблоны для менеджера и клиента.
- Искать фото в таком порядке:
  1. прямые ссылки из файла;
  2. изображения на странице товара поставщика;
  3. поисковая выдача по артикулу и сайту поставщика;
  4. общий интернет-поиск по артикулу и названию.
- Скачивать найденные изображения в локальное хранилище.
- Запоминать товары и изображения в SQLite.
- Формировать ZIP-архивы под клиента по шаблонам именования.
- Поддерживать отдельные профили клиентов в `data/client_profiles.json`.

## Текущий поток работы

1. Менеджер загружает файл товаров в раздел `Каталог фото`.
2. Сервис пытается собрать максимум фото по каждому товару.
3. Найденные изображения сохраняются локально и связываются с артикулом.
4. В разделе `Клиенты` загружается файл соответствия:
   - `артикул` товара;
   - клиентский код для именования файла.
5. Сервис собирает ZIP-архив и переименовывает фото по правилам клиента.

## Поддерживаемые входные колонки

Обязательные:

- `артикул`
- `название`

Опциональные:

- `шт`
- `сайт`
- `артикул поставщика`
- `ссылка на товар`
- `картинки`
- `комментарий`

Для поля `картинки` поддерживается разделитель `;`, а также `,` и перенос строки.

## Клиентские профили

Правила лежат в `data/client_profiles.json`.

По умолчанию добавлены:

- `sportmaster`
- `detmir`

### Спортмастер

- Имя файла: `Код цветомодели_n`
- Форматы: `jpg`, `jpeg`, `png`
- Ограничение размера: до `50 МБ`

Источник, проверен 27 апреля 2026 года:

- https://seller-help.sportmaster.ru/pages/viewpage.action?pageId=48365664

### Детский Мир

- Имя файла: `штрихкод_номер-фото`
- Форматы: `jpg`, `jpeg`, `png`, `webp`
- Длинная сторона: `1000-8000 px`
- Ограничение размера: до `10 МБ`

Источник, проверен 27 апреля 2026 года:

- https://help.detmir.market/edit-media

## Быстрый старт

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
node scripts/build_templates.mjs
streamlit run app.py
```

## Готовые шаблоны Excel

После запуска генератора шаблоны лежат здесь:

- `templates/manager_import_template.xlsx`
- `templates/client_mapping_template.xlsx`
- `templates/sportmaster_upload_template.xlsx`
- `templates/detmir_upload_template.xlsx`
- `templates/sportmaster_links_template.xlsx`
- `templates/detmir_links_template.xlsx`

Что внутри:

- `manager_import_template.xlsx`:
  рабочий лист `Менеджер` с заголовками в первой строке, файл можно сразу заполнять и загружать обратно в UpdatPic;
  лист `Как использовать` с подсказками по полям.
- `client_mapping_template.xlsx`:
  рабочий лист `Клиент` с колонками `article` и `client_code`;
  лист `Как использовать` с примерами под Спортмастер и Детский Мир.
- `sportmaster_upload_template.xlsx`:
  рабочий лист `Спортмастер` с колонками `article` и `Код цветомодели`;
  лист `Как загружать` с примером итоговых имён файлов `КодЦветомодели_1`.
- `detmir_upload_template.xlsx`:
  рабочий лист `Детский Мир` с колонками `article` и `Штрихкод товара`;
  лист `Как загружать` с примером итоговых имён файлов `Штрихкод_01`.
- `sportmaster_links_template.xlsx`:
  для сценария без парсинга, когда уже есть готовые прямые ссылки;
  колонки `article`, `Код цветомодели`, `Ссылки на фото`.
- `detmir_links_template.xlsx`:
  для сценария без парсинга, когда уже есть готовые прямые ссылки или публичные ссылки Яндекс Диска;
  колонки `article`, `Штрихкод товара`, `Ссылки на фото`.

## Важные ограничения текущей версии

- Это стартовый каркас, а не финальный production-парсер.
- Поиск по интернету сейчас идёт через HTML-выдачу DuckDuckGo и разбор найденных страниц; для крупных объёмов лучше заменить на управляемый поисковый провайдер/API.
- Нет автоматической модерации качества фото по правилам маркетплейсов; сервис нормализует формат, имя и базовые размеры, но контент всё равно нужно проверять глазами.
- Нет фонового воркера, очередей и многопоточности.
- Нет авторизации пользователей и разграничения по клиентам.
